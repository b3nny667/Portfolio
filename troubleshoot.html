GitHub Pages Troubleshooting Guide

Follow these steps to diagnose and fix issues with your portfolio website

1. Verify GitHub Pages Configuration

Check if GitHub Pages is enabled

Go to your repository on GitHub → Settings → Pages

Under "Source", it should be set to: Deploy from a branch

And the branch should be: main (or master) and folder: / (root)

Check the deployment status

Go to your repository on GitHub → Actions

You should see a workflow called "pages-build-deployment"

If there's a red ❌, there's an error. If there's a green ✅, it deployed successfully.

2. Verify Your Repository Structure

Your repository should have these files at the root:

index.html style.css script.js send_email.php /assets (folder) /certificates (folder) /projects (folder)

Check your image paths

Make sure all image paths in your HTML are correct:

<!-- Certificates --> <img src="certificates/Tectigon Academy-1.png"> <!-- Project images --> <img src="projects/crypto app/img/crypto app.PNG">

3. Common Issues and Fixes

GitHub Pages not building

Sometimes GitHub Pages fails to build if there are errors in your code.

Check the Actions tab for any error messages.

File name case sensitivity

GitHub is case-sensitive. Make sure your file references match exactly:

<!-- This will work --> <img src="certificates/Tectigon Academy-1.png"> <!-- This might NOT work --> <img src="Certificates/Tectigon Academy-1.png">

Spaces in file names

Spaces in file names can cause issues. Consider renaming:

"crypto app.PNG" → "crypto-app.PNG"

Then update your HTML accordingly.

4. Quick Fixes to Try

Rename your main HTML file

GitHub Pages looks for index.html by default. Make sure your main file is named exactly:

index.html

Check for configuration files

Add a .nojekyll file to the root of your repository to disable Jekyll processing:

# Create an empty file named .nojekyll # This tells GitHub Pages not to use Jekyll

Wait for deployment

GitHub Pages can take up to 10 minutes to deploy after you push changes.

Check the status at: https://github.com/b3nny667/b3nny667.github.io/actions

5. Final Checklist

  • GitHub Pages is enabled in repository settings
  • All files are in the main branch (not a separate branch)
  • Main HTML file is named index.html
  • All file paths are correct and case-sensitive
  • No build errors in the Actions tab
  • You've waited at least 5 minutes after your last commit
  • You're visiting the correct URL: https://b3nny667.github.io
